# 小红书 第0期 v2 (修正版)

## 标题
6个AI盲评，结构化读源码惨败

## 正文

JSON schema + 验证门禁 + 架构角色，精心设计的结构化流程，被零流程的 baseline 打爆了💥

🧪 实验设计
Claude Code 51万行公开源码，6个 Sonnet agent 分头读核心模块，跑两轮：
① 自由 Markdown
② 结构化 JSON
匿名后让独立 Opus 盲评打分

📊 结果：自由文本 6:1 完胜

结构化只赢了"证据质量"（+1.33）
但输了"洞察深度"（-1.00）和"实用性"（-1.17）

JSON 把 agent 变成了索引机器——能检索，但说不出"这个代码库本质上是个XX"这种改变认知的判断

💡 结论：结构帮验证，自由帮洞察。别用同一个格式

✅ v3 方案：先 JSON 提取（Haiku，便宜）→ 再自由文本解读（Opus）。分开跑，分开把关

完整实验数据 + 代码已开源，评论区见链接👇
